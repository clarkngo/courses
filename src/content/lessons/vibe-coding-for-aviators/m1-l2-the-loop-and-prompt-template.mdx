## 🔁 The Vibe Coding Loop

Every project you build this week — today's boarding pass page, tomorrow's quiz, Friday's final build — runs through the same five-step loop. Learn it once, reuse it forever.

<LoopDiagram
  label="The vibe coding loop: Describe, Build, Test, Refine, Repeat, arranged in a circle with arrows connecting each step back to the next."
  steps={[
    { title: '1. Describe', subtitle: 'say what you want' },
    { title: '2. Build', subtitle: 'Gemini writes it' },
    { title: '3. Test', subtitle: 'click it yourself' },
    { title: '4. Refine', subtitle: 'ask for one fix' },
    { title: '5. Repeat', subtitle: 'until it feels right' },
  ]}
/>

**Describe** what you want in plain English. **Build** — Gemini writes real code and shows you a live preview. **Test** it yourself: click every button, try to break it. **Refine** by asking Gemini for one specific fix or addition. **Repeat** the Test → Refine cycle until it feels right — usually three or four laps, not one.

The loop never really ends. Even the projects you "finish" on Friday could keep looping forever. You just decide when to stop.

## 📝 The Prompt Template

A blank prompt box is intimidating. A fill-in-the-blank sentence isn't. Use this template every time you start something new this week:

<CodeLab title="Fill-in-the-Blank Prompt Template">

```
Build a [thing] that does [core feature], for [who it's for],
styled like [vibe/mood]. Make [X] happen when I [click/type/etc.].
```

**Today's example** — a personal boarding-pass profile page:

```
Build a boarding-pass-style webpage that shows my name, home
airport, dream destination, and a fun personal fact, for
someone meeting new classmates for the first time, styled like
a sleek airline check-in display with a navy and gold color
scheme. Make a small paper-airplane icon animate across the
screen when I click "Check In."
```

Notice every blank is filled with something *specific* — not "make it cool," but "navy and gold," not "something happens," but "a paper-airplane icon animates across the screen." Specific blanks produce specific code. Vague blanks produce Gemini's best guess, which is rarely what you pictured.

</CodeLab>

<Pitfall title="The Kitchen-Sink Prompt">

New builders often try to describe *everything* in one giant first prompt — five features, three animations, a color scheme, and a login system. Gemini will attempt all of it, and when something's off, you won't know which part broke it.

Start with the template as written — one core feature, one interaction. Add the second feature as a *refine* step, once the first one is working. One change per loop keeps you in control.

</Pitfall>

**Refine with one specific fix — don't start over with a whole new prompt.**

Something like: *"Move my name to the top center, and change the background to navy blue instead of gray."* That's a Refine step: it names exactly what's wrong and exactly what you want instead. Retyping the entire original prompt from scratch throws away a build that was already 90% right, and gives Gemini no signal about what specifically needs to change.
