## Why Active Learning Works

Passive reading gives you a false sense of understanding.

The moment you close the tab, 70% of it evaporates. Active recall — writing code, answering questions, explaining concepts back — is what turns exposure into mastery.

That's why every lesson here ends with a hands-on exercise and a quiz. It's not busywork. It's the science of learning.

<CodeLab title="Set Up Your Workspace">

Let's make sure your environment is ready before we dive in.

**Step 1** — Open your terminal and verify Node.js is installed:

```bash
node --version
# Should print v18.0.0 or higher
```

**Step 2** — Clone the starter repository:

```bash
git clone https://github.com/your-org/course-starter.git
cd course-starter
npm install
```

**Step 3** — Start the dev server:

```bash
npm run dev
# Open http://localhost:4321 in your browser
```

🎉 You should see the starter app running. You're ready for Lesson 2!

</CodeLab>

<Quiz question="What is the primary benefit of active learning over passive reading?">

<div>

**Active learning works because it forces retrieval.**

When you write code or answer a question, your brain has to reconstruct the knowledge — not just recognize it. This retrieval process strengthens the neural connections that make recall fast and reliable.

Passive reading feels productive but is largely recognition, not retrieval. You feel like you understand because the answer is right there on the page.

**Bottom line:** doing > reading. Always run the code.

</div>
</Quiz>
